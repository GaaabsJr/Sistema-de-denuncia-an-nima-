## 📚 Sobre o Projeto

Observa-se, no contexto social e institucional, uma demanda crescente por mecanismos seguros e acessíveis que permitam a qualquer cidadão reportar irregularidades, crimes, corrupção ou situações de risco sem que precise se identificar, evitando assim represálias e incentivando a participação ativa na construção de ambientes mais justos e transparentes.

O objetivo deste projeto é desenvolver uma plataforma web de denúncia anônima capaz de receber, organizar e encaminhar relatos de forma segura e confidencial, garantindo o anonimato do denunciante e permitindo que órgãos competentes ou gestores institucionais acompanhem e tratem cada ocorrência registrada com eficiência e responsabilidade.

---

## 📋 Requisitos Funcionais

- **RF-01:** Permitir o registro de denúncias anônimas, sem exigir identificação do usuário.
- **RF-02:** Permitir que o denunciante escolha a categoria da denúncia (corrupção, assédio, violência, irregularidades, etc.).
- **RF-03:** Permitir o anexo de arquivos de evidência (imagens, documentos ou vídeos) à denúncia.
- **RF-04:** Gerar um código único de protocolo ao final de cada denúncia registrada, permitindo o acompanhamento posterior.
- **RF-05:** Permitir que o denunciante consulte o status de sua denúncia por meio do código de protocolo, sem necessidade de cadastro.
- **RF-06:** Permitir o cadastro e autenticação de administradores/gestores responsáveis pelo tratamento das denúncias.
- **RF-07:** Permitir que administradores visualizem, classifiquem, atualizem o status e adicionem comentários internos às denúncias recebidas.
- **RF-08:** Permitir que administradores encaminhem denúncias para outros setores ou órgãos responsáveis dentro do sistema.

---

## ⚙️ Requisitos Não Funcionais

- **RNF-01:** A plataforma deve garantir o anonimato do denunciante, não armazenando dados pessoais identificáveis sem consentimento explícito.
- **RNF-02:** O sistema deve utilizar criptografia no armazenamento e na transmissão de dados sensíveis.
- **RNF-03:** A interface deve ser simples, intuitiva e responsiva, acessível em computadores e dispositivos móveis.
- **RNF-04:** O sistema deve apresentar tempo de resposta adequado para o registro e consulta de denúncias.
- **RNF-05:** A documentação do projeto deve ser mantida de forma organizada para facilitar o trabalho em equipe.
- **RNF-06:** O código deve seguir padrões de organização, legibilidade e manutenção definidos pela equipe.
- **RNF-07:** A plataforma deve permitir evolução futura para integração com órgãos externos, notificações e painéis estatísticos.
